# Variables de decision
arc[i,j,k] : BoolVar
  # drone k emprunte arc i→j ?

# Objectif
Minimize( Σ arc[i,j,k] × dist[i,j] )
MILP + propagation de contraintes — bien plus efficace qu'une recherche exhaustive.
03 — Modelisation

Contraintes CP-SAT

Couverture clients
∀i client : Σj,k arc[i,j,k] = 1
Chaque client est visite exactement une fois par un seul drone.
Conservation du flux
∀k, ∀j : Σ entrants = Σ sortants
Garantit des routes coherentes — pas de drone qui disparait.
Batterie (style MTZ)
battery[j] = battery[i] − dist(i,j)
battery[i] ≥ dist(i, depot)
Propagation arc par arc. Retour au depot toujours garanti.
Capacite poids + volume
load[j] = load[i] + demand[j]
vol[j] = vol[i] + volume[j]
Accumulation le long de la route, bornee par max_load et max_volume.
Les contraintes de batterie et capacite utilisent OnlyEnforceIf(arc[i,j,k]) pour n'etre actives que si l'arc est emprunte.
04 — Pathfinding

Contournement des Zones NOTAM

CP-SAT a besoin d'une matrice de distances entre tous les noeuds. On ne peut pas utiliser la distance euclidienne si une zone NOTAM est sur le chemin.
1

Tentative directe

On verifie si le LineString entre deux points intersecte un polygone NOTAM. Si non, distance euclidienne directe.

2

Dijkstra sur grille de waypoints

Grille de points surs generee hors des zones. Dijkstra trouve le chemin le plus court evitant les obstacles.

3

Matrice finale

Les distances reelles alimentent CP-SAT. Les chemins complets sont conserves pour l'affichage sur la carte.

# 1. Chemin direct ?
if is_safe(p1, p2):
    return dist_euclidienne, [p1, p2]

# 2. Dijkstra via waypoints
queue = [(0, start, [p1])]
while queue:
    d, curr, path = heappop(queue)
    if curr == end:
        return d, path
    for nxt, w in adj[curr]:
        heappush(queue, (d+w, nxt, ...))
